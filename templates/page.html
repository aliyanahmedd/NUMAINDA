About

Automated Open Source Intelligence

OSINT Agent is a multi-agent reconnaissance platform that gathers, correlates, and analyzes publicly available intelligence about any domain, IP address, or email - then produces a risk-scored report powered by Claude AI.

What it does

Give it a target and it autonomously runs 14 intelligence phases across dozens of free and commercial data sources. It maps the digital footprint, identifies the attack surface, checks for leaked data and known vulnerabilities, and assigns a 0-10 risk score. The kind of work that would normally take an analyst hours, done in minutes.

How it works

A central orchestrator coordinates specialized sub-agents, each responsible for one domain of intelligence such as DNS, email discovery, subdomain enumeration, breach checks, and threat intel. Findings are aggregated, scored by a weighted risk model, and passed to Claude Sonnet for a final analyst-grade report.

Built with

  • Python + Flask: backend orchestration and web interface
  • Anthropic Claude (Sonnet 4.6): AI correlation and reporting
  • SQLite: local persistence of every scan
  • Plotly + NetworkX: interactive network graphs
  • Leaflet.js: IP geolocation maps

Ethical use

This tool is for authorized reconnaissance only: domains you own, targets you have written permission to test, bug bounty scope, or security research. Unauthorized use may violate CFAA, GDPR, PECA, and other laws.

API

HTTP Endpoints

The Flask app exposes simple HTTP routes for launching scans, streaming progress, and retrieving results. All routes are served from the local instance.

Scanning

MethodEndpointDescription
POST/scanStart a scan (form field: target)
GET/scan/<id>Live progress page
GET/scan/<id>/streamSSE progress stream
GET/scan/<id>/resultsResults dashboard
GET/scan/<id>/graphNetwork graph (HTML)
GET/scan/<id>/download/<type>Download report (html/txt/json/pdf)

Records

MethodEndpointDescription
GET/recordsAll stored scans
GET/records/<id>Stored scan detail
GET/records/<id>/graphStored network graph

Example

Launch a scan with curl:

# start a scan
curl -X POST http://127.0.0.1:5000/scan \
     -d "target=example.com" -L

# results are saved to output/reports/ and the SQLite db

Or call the orchestrator directly in Python:

from agents.orchestrator import OSINTOrchestrator

orch = OSINTOrchestrator()
findings = orch.run("example.com")
print(findings["scores"]["total"])

Documentation

Using the Scanner

Everything you need to run scans, read results, and understand what the numbers mean.

What you can scan

Type any of these into the search bar on the home page:

TypeExampleWhat runs
Domainexample.comFull 14-phase scan
Emailuser@example.comExtracts the domain, full scan
IP address8.8.8.86 IP-focused phases

The scan runs live, so you'll watch each phase stream in the console, then land on the results dashboard automatically when it finishes (usually 1 to 3 minutes).

Reading the dashboard

  • Risk ring: the headline 0-10 score; color-coded green / yellow / red
  • Stat cards: quick counts: emails, subdomains, breaches, malicious indicators
  • Module cards: full detail per phase (DNS, ports, CVEs, geolocation, etc.)
  • 📍 Map buttons: open an interactive map pin for any discovered IP
  • AI Analysis: Claude's executive summary and prioritized recommendations

Reading the risk score

  • 0-4 Low: minimal exposed attack surface
  • 4-7 Medium: review the flagged findings
  • 7-10 High: significant exposure, act on the recommendations

The overall score is a weighted average across breaches (25%), subdomains (20%), emails (15%), CVEs (10%), threat intel (10%), SSL (8%), ports (7%), and pastes (5%).

Saving & revisiting scans

Every scan is stored automatically. Open Records to see your full scan history with risk scores and finding counts. Click View Details on any row to reopen the complete dashboard, network graph, and AI analysis. No need to re-scan.

Exporting results

  • Network graph: interactive map of the attack surface
  • HTML / Text / JSON: downloadable reports from any results page
  • PDF: print-ready report (live scans)

Why a category shows 0

A 0 means "nothing found by the data sources we query," not a guarantee of safety. Many sources only return publicly indexed data and some have free-tier limits. For example, 0 emails usually means none are publicly listed, not that the organization has none. Treat findings as a starting point for investigation.

Tips

  • Start with a domain for the richest results, since it feeds every phase
  • Use the example chips on the home page to try it instantly
  • Scanning takes longer for large domains with many subdomains, which is normal
  • Only scan targets you own or are authorized to test
